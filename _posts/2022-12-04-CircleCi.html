For the next portion of the website, we used circle ci. Circle Ci is a program that tracks each commit to make sure that the project can build and deploy.

Through this application you can also filter whose builds you are seeing and what they have done. This is good for businesses who have a lot of engineers modifying files. It is also good for administrators to see what progress an employee is doing.

In my opinion, the best way to use circle ci is for debugging. It is much easier to see what errors are happening and what can be done to fix it compared to a regular IDE. Hopefully in the future I could use an application like this to check my works.

Circle CI's user interface is especially good for analyzing why your build failed, and seeing important statstics during the process for your project.

This application used with github and VS Code makes creating a project that less stressful.
